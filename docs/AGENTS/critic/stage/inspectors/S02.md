## 3. 内容审查要点

- [ ] 检索策略是否覆盖多个数据库？
- [ ] 文献筛选标准是否明确？
- [ ] 文献分类是否有清晰的逻辑？
- [ ] 是否识别了至少 3 个研究空白 (Gap)？
- [ ] 每篇关键文献是否有局限性分析？
- [ ] **是否建立了方法论/技术方案库（§5）？**
- [ ] **方案库是否包含可用于解决 Gap 的具体方法/架构/模块？**
- [ ] **方案库是否为每个 Gap 提供了至少 1 个潜在技术路线？**
- [ ] 参考文献列表是否完整（含 BibTeX）？

---

## 4. 常见错误模式

- 只搜索了一个数据库
- 只读了 abstract 没读 method
- Gap 识别流于表面，没有文献支撑
- 遗漏了领域的经典/奠基性工作
- **只识别了 Gap，没有调研"用什么方法可以解决这些 Gap"**
- **方案库中的方法与我们识别的 Gap 不匹配**

---

## 5. 跨 Stage 一致性检查

- [ ] 该 Stage 引用的上游文档是否存在？
- [ ] 引用的上游数值是否与源文档一致？
- [ ] 该 Stage 的结论是否与上游文档矛盾？
- [ ] 下游 Agent 能否仅凭本 Stage 产出 + 上游文档完成工作？
