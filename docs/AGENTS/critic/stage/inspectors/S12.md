## 3. 内容审查要点

- [ ] 实验策略是否明确？（迭代预算、收敛条件）
- [ ] Baseline 结果是否已记录并合理？
- [ ] 迭代循环记录（results.tsv）是否完整？每次迭代是否有 commit hash、指标、说明？
- [ ] 迭代方向判断是否有依据？（不只是表面指标对比）
- [ ] 所有 git commit 是否保留？（未被 reset）
- [ ] 收敛条件是否被正确应用？
- [ ] 最优 checkpoint 是否已保存并记录路径？
- [ ] 环境信息（Python/PyTorch/CUDA/硬件）是否完整记录？
- [ ] 遇到的问题与解决方案是否记录？
- [ ] **不做 KEEP/DISCARD/FIX 最终决策**（此为 S13 Analysis Agent 职责）

---

## 4. 常见错误模式

- 迭代次数过少（<3 次）未充分探索
- 每次迭代仅做表面微调（学习率、batch size）而非方法级修改
- 最优结果对应的 commit hash 未记录
- 结果未完整保存（只记了平均值）
- 在 S12 内执行了本应属于 S13 的最终评估决策
- 未固定随机种子导致结果不可复现
- 简化配置运行（pilot mode）未说明理由和回归完整配置的计划
