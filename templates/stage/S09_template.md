# S09: Baseline Selection

> Agent: `method`
> Phase: P2
> 目标: 选择公平、有代表性的对比 baseline，确保实验结论可信

---

## 1. 核心目标

从 S02 的文献综述（尤其是 §3.2 对比矩阵和 §6.1 逐论文实验细节）中识别关键竞争方法，选择能公平验证我们方法有效性的 baseline。本阶段的核心任务是：
- **全面覆盖**: baseline 应覆盖主要技术路线
- **公平对比**: 确保所有方法在相同条件下比较（参考 S02 §6 的训练协议惯例）
- **无冲突**: baseline 选择与消融设计不冲突
- **Subagent 检验**: 验证 baseline 实现的可行性
- **性能基准**: 从 S02 §6.1 中提取各 baseline 的已知性能，作为实现的 sanity check

**选择依据**:
- 必须包含领域广泛使用的标准方法
- 必须包含最近 1-2 年的 SOTA（如有）
- 应覆盖不同技术路线（与我们方法形成对比）
- 从 S02 的文献分类中选取代表性工作

---

## 3. 公平性保证

- [ ] 所有方法使用相同的数据划分
- [ ] 所有方法使用相同的评估指标
- [ ] 计算资源保持一致（相同 GPU 型号）
- [ ] 超参数调优策略一致（相同搜索空间/调优轮数）
- [ ] 训练预算一致（相同 epoch 数或时间上限）
- [ ] 代码框架一致（相同的数据加载、预处理流程）

---

## 5. 潜在问题与应对

| 问题 | 影响 | 应对方案 |
|------|------|---------|
| 某些 baseline 官方代码不可用 | 需要自行复现，引入误差 | 优先选择有官方代码的方法；如必须复现，记录复现细节 |
| 某些 baseline 超参数敏感 | 公平性受损 | 使用与我们的方法相同的调优策略 |
| 某些 baseline 计算成本过高 | 无法在预算内完成 | 选择轻量级版本或排除 |
| 某些 baseline 与我们的方法有重叠 | 对比不够清晰 | 明确区分核心差异 |
