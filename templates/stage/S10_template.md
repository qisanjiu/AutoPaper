# S10: Full Experiment Plan

> Agent: `method`
> Phase: P2
> 目标: 整合完整的实验计划，覆盖方法 + baselines 的全部实验流程

---

## 1. 核心目标

将 S06-S09 的所有设计决策整合为一个可执行的完整实验计划。本阶段的核心任务是：
- **计划整合**: 将方法实验和 baseline 实验整合为统一执行计划
- **执行顺序**: 明确实验的先后依赖关系
- **成功/失败标准**: 定义主实验的成功判定标准
- **风险评估**: 识别执行风险并制定应对策略

> **注意**：消融实验的具体设计（组件选择、变量隔离方案、预期效果）在 P4 S14 基于 S13 实际验证结果完成。本阶段仅为消融实验做调度层面的预留（资源预算、在决策树中的位置）。

---

---

## 3. 执行顺序与决策树

```
Exp-1 (主实验: 完整方法 vs baselines)
  ├── 如果成功（方法优于 baselines，S13 KEEP 决策）
  │   └── 推进到 P4 (S14 消融设计 → S15 消融代码 → S16 消融执行 → S17 消融分析)
  │
  └── 如果失败（方法不优于 baselines）
      └── S13 决策（FIX → S12 / BACKTRACK → S11 / S06 / S08 / S04）
      （无论主实验成功与否，S13 均为 Phase 3 唯一决策点）
```

---

## 4. 成功/失败判定标准

### 4.1 主实验成功标准
- [ ] 方法在主要指标上优于最强 baseline
- [ ] 统计显著性达到 p < 0.05
- [ ] 多 seed 结果稳定（方差合理）

### 4.2 消融实验成功标准

> 消融实验的成功标准将在 P4 S14 中根据 S13 实际验证结果定义，不在本阶段预设。S10 仅为其预留资源（见 §5 资源预算）。

### 4.3 触发回溯的条件
- [ ] 方法显著弱于 baseline → 回溯到 S06 或 S11
- [ ] 结果不可复现 → 回溯到 S11
- [ ] 假设被否定 → 回溯到 S04

## 6. 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 实验运行时间超预期 | 中 | 预算耗尽 | 设置单次时间上限；准备简化方案 |
| 实现 bug 导致结果无效 | 中 | 时间浪费 | 代码 review；先运行小规模 pilot |
| 数据下载失败或超时 | 中 | 实验无法使用真实数据 | 遵循 S07 §6.3 降级策略；合成数据需明确标注 |
| 数据加载问题 | 低 | 实验中断 | 在 P0 阶段提前验证数据管道 |
| 环境依赖冲突 | 低 | 无法运行 | 使用 conda/venv 隔离环境 |
