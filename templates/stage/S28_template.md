# S28: Methodology Section

> Agent: `writing`
> Phase: P7
> 目标: 撰写论文的方法部分，让读者能复现我们的方法

---

## 1. 核心目标

将 S06 的方法设计转化为符合学术规范的方法论 section。本阶段的核心任务是：
- **遵循框架**: 根据 S26 §3 的领域框架决策（是否有 Preliminaries、Method 命名等）组织内容
- **清晰描述**: 让读者理解方法的工作原理
- **可复现**: 提供足够的细节让同行可以复现
- **结构化**: 按照 S26 的大纲组织内容
- **术语统一**: 遵循 S26 §5 术语表和符号表
- **Anti-Leakage**: 所有 LLM 写作调用前附加 Anti-Leakage Prompt

---

## 2. 写作结构

### 2.1 问题定义 (Problem Formulation)
- 输入/输出定义
- 目标函数（数学形式）
- 与领域标准问题的关系

### 2.2 方法概述 (Method Overview)
- 一句话概括核心思想
- 架构图引用（Fig 1，由 S32 生成）
- 与已有方法的高层次对比

### 2.3 核心组件
按 S06 的设计逐个描述：

#### 组件 A
- **目的**: 解决什么子问题
- **设计**: 具体实现
- **公式**: 数学表达

#### 组件 B
- ...

### 2.4 算法流程
- 伪代码或算法框（必须包含）
- 复杂度分析（如有）

### 2.5 理论分析（如有）
- 收敛性/正确性证明
- 复杂度分析

---

## 3. 写作原则

- **术语一致性**: 所有核心术语使用 S26 §5.1 术语表中定义的统一表述，首次出现时定义缩写
- **符号清晰**: 所有数学符号使用 S26 §5.2 符号表中定义的统一符号，首次出现时定义
- **渐进式描述**: 从简单到复杂，从高层到细节
- **对比明确**: 与已有方法的区别在描述中自然体现

## 6. 验证与检查

- [ ] 方法描述足够详细，同行可复现
- [ ] 所有符号和术语已定义
- [ ] 伪代码/算法框正确
- [ ] 与 S06 设计一致
- [ ] Section 命名和结构与 S26 §3 领域框架决策一致
- [ ] 术语与 S26 §5.1 术语表一致（非仅与 S27 一致）
- [ ] Anti-Leakage Prompt 已应用
- [ ] Academic Style Prompt 已应用
- [ ] 长度在 S26 §4 页数预算内
- **学术风格自查**：
  - [ ] 无 contractions（don't, can't, won't, it's, we're 等）
  - [ ] 无博客式问句标题
  - [ ] 无口语化过渡词
  - [ ] 正文无 bullet list
  - [ ] 无单句段落（数学定义框/算法框后的过渡除外）
  - [ ] 所有结论性 claim 有适当 hedging
  - [ ] 无 informal emphasis（"a lot of", "really", "very", "huge"）
  - [ ] 引用格式统一使用 `\cite{key}`
  - [ ] 数学符号与 S26 §5.2 符号表一致
